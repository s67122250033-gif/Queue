FIFO (First-In, First-Out): หมายความว่า ข้อมูลที่เข้ามาเป็น "ลำดับแรก" จะต้องเป็นข้อมูลตัว "แรก" ที่ถูกนำออกไปปฏิบัติงานเสมอ เหมือนการเข้าแถวซื้อของ

การประยุกต์ใช้ในระบบคิวโรงพยาบาล: ผู้ป่วย P001 มาลงทะเบียนก่อนใครเพื่อน เมื่อเจ้าหน้าที่เรียกคิว (poll() หรือ remove()) P001 และ P002 จึงได้รับการบริการตามลำดับ ส่วนผู้ป่วยที่มาทีหลังอย่าง P006 และ P007 จะถูกนำไปต่อท้ายคิว (add()) เพื่อรอรับบริการถัดไป

Time Complexity เบื้องต้น: การนำเข้าข้อมูลท้ายคิว (Enqueue) และการนำข้อมูลออกหน้าคิว (Dequeue) มี Time Complexity เป็น $O(1)$ เนื่องจากระบบอ้างอิงตำแหน่งหัว (Head) และท้าย (Tail) คิวได้โดยตรง
